# Academic Job Search App — Implementation Plan

## What This Is
A lightweight, dark-mode static web app hosted on GitHub Pages that twice-daily scrapes
University Affairs (universityaffairs.ca — the primary Canadian academic job board) for
real teaching job postings matching your profile, then displays them organized by province
with match scoring.

## Data Flow

```
GitHub Action (6am + 6pm EST)
  → runs scraper.py
    → fetches University Affairs job board (keyword searches)
    → parses: title, institution, province, deadline, URL
    → scores each job (Strong / Partial / None)
    → filters to Canada only
    → writes jobs.json
  → commits & pushes jobs.json to repo
  → GitHub Pages auto-deploys (no build step needed)
User visits site
  → app.js fetches jobs.json
  → renders cards grouped by province (ON + BC first, rest behind Expand button)
```

---

## Scraper Strategy

**Source:** `https://www.universityaffairs.ca/career-centre/job-board/`

University Affairs has a searchable job board. The scraper will:
1. Run multiple keyword searches (neuroscience, anatomy, kinesiology, etc.)
2. Deduplicate by job URL
3. Extract: job title, institution, city/province, deadline, application link
4. Province is parsed from location text → mapped to standard province name
5. Match score assigned based on title + description keyword matching

Rate limiting: polite delay between requests, proper User-Agent header.

If University Affairs provides an RSS feed (they sometimes do), we use that instead
as it's more reliable and scrape-friendly.

## Frontend Features

- **Always dark mode** — deep background (#0f1117), muted card backgrounds, clean typography
- **Province sections** — Ontario and BC rendered first, expanded by default
- **"Show other provinces" button** — reveals AB, MB, SK, QC, NS, NB, PEI, NL, territories
- **Job cards** show:
  - Job title (linked to posting)
  - Institution name
  - City, Province
  - Deadline (color-coded: red if < 2 weeks, yellow if < 4 weeks)
  - Direct application link button
  - Match badge: `STRONG MATCH` (green) or `PARTIAL MATCH` (yellow)
- **Last updated** timestamp shown at top
- **"No jobs found"** fallback if scraper data is stale or empty

## GitHub Actions Workflow

```yaml
# Runs at 6am and 6pm EST (11:00 and 23:00 UTC)
schedule:
  - cron: '0 11 * * *'
  - cron: '0 23 * * *'
```

The Action:
1. Checks out repo
2. Sets up Python 3.11
3. Installs `requests` + `beautifulsoup4`
4. Runs `scraper.py`
5. Commits `jobs.json` if changed (with message "chore: refresh job listings [timestamp]")
6. Pushes to main
